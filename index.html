      📋 排班規則說明

      👤 值班　第一優先
      值班人員依照人員名單順序，平日與假日各自獨立輪續，若有人請長假則由下一位遞補，衝突不受影響，第一優先順位。
      📋 支援　第一優先
      每週四工作日安排，按照非護理師名單順序輪替，與值班完全獨立。與值班衝突可同時存在。
      🩹 卡介苗　第一優先
      當月第一個週二工作日，由特別資格護理師輪替，擔任卡介苗的人員當日不會排其他門診相關職務，可與值班同時存在。
      🏥 門診 / 掛號 / 前台 / 預登 / 注射
      每週二、四工作日按班別需求安排，以全年累積次數均勻為原則，次數少的人優先被安排，確保平衡。
      🚫 停班2線
      每天安排，平日與假日分別排序，若遇值班衝突，系統會自動與最少換過的下一位平／假日調換，直到同日足兩個人力為止。
      💡 若有人員請長假或特殊不排班，可在管理後台「🗓 自動排班 → 排班排除設定」中設定，系統會自動跳過該人員並由下一位遞補。

      📋 門診相關排班邏輯說明
      ① 全局補正優先：以護理師的「門診類合計次數（本年度累積）」最少者優先分配，確保月平均偏低者自動多值。
      ② ±2 上限截斷：若某人合計次數已超出當月候選者平均 2 次以上，排序鍵視為相等，改由 ③ 決定，防止代償過頭。
      ③ 職務欄均衡：同等優先序下，先補足個別職務（門診/掛號/前台/預登…）次數最少的欄位，使各欄分布均勻。
      ④ 輪轉 Round-Robin：完全相同時，以當日排班序號輪轉起點，確保不固定由同一人先被選到。
      ⑤ 同日不重複：同一天已被排入任一門診欄的人員，當天不再出現於其他門診欄。
      補償值 = 截至該月累積次數 − 同期期望次數
      正值（橘色）= 已超出期望，下個月排班優先序降低；
      負值（綠色）= 尚未達期望，下個月排班優先序提升。
      ● 原始 = 當月實際排班次數　● 累積 = 截至該月的合計次數
      ● 補償值 = 累積次數 − 同期期望次數（正 = 超出，負 = 待補）
